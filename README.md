# FinTwit Prediction Audit

**Do financial influencers on X actually beat the market — and is anyone genuinely skilled, or just lucky?**

A fully reproducible, LLM-powered statistical audit of ~18,000 directional stock predictions made by 51 FinTwit accounts between January 2021 and February 2025.

---

## Research Questions

**RQ1 — Accuracy & benchmark.** How accurate are FinTwit predictions directionally, and do they beat a passive SPY benchmark over the matched prediction horizon?

**RQ2 — Skill vs luck.** After correcting for multiple comparisons, is any account's track record statistically distinguishable from chance? Uses empirical-Bayes shrinkage to separate genuine skill from sampling noise across 51 accounts.

**RQ3 — Signal value.** Do tweet-level signals — sentiment, trade type, horizon, engagement, LLM confidence — carry predictive information? Includes a formal calibration analysis of the LLM's self-reported confidence score and a predictive model of `prediction_correct`.

---

## Key Findings

| Metric | Value |
|---|---|
| Validated directional predictions | 10,690 |
| Directional accuracy | **45.0%** (95% CI [44.1%, 46.0%]) — significantly below 50% |
| beats_market rate | **44.4%** (95% CI [43.5%, 45.3%]) — significantly below SPY |
| Accounts surviving BH-FDR correction | **9 of 51** credibly above 50% |
| Accounts surviving Bonferroni correction | **1 of 51** (account_19) |
| LLM confidence score vs accuracy correlation | **r = −0.0005** (p = 0.96) — completely uncalibrated |
| Predictive model AUC (gradient boosting) | **~0.70** (driven by RQ1 base rates, not novel features) |
| Cross-regime persistence test | **Not computable** — 99.2% of validated predictions fall in 2023-2025; only 2 accounts qualify for split-half test |
| Worst-performing segment | High-volume bullish explicit trade calls: **38.0% correct**, 38.5% beats market |

**Contrarian takeaway:** The tweets most likely to make a retail investor act — explicit buy calls, bullish, from a large account, with high LLM confidence — underperform random by ~12 percentage points.

## How to Reproduce

**Prerequisites:** Python 3.11+, OPENAI_API_KEY in a `.env` file, source data at `OLD project/validated_predictions.csv`.

> **Note:** The raw tweet scrape is not redistributed (X ToS), so the pipeline is not end-to-end runnable without that input. The code and `Statistical_Report.pdf` are provided for inspection and reproducibility review.

```bash
# 1. Create and activate a virtual environment
python -m venv .venv
.venv\Scripts\activate          # Windows
# source .venv/bin/activate     # macOS/Linux

# 2. Install dependencies
pip install -r requirements.txt

# 3. Build the raw dataset and pseudonymise accounts
python src/ingest.py

# 4. (Optional) Run individual phase scripts to inspect intermediate output
python scripts/run_rq1.py      # RQ1 accuracy & benchmark stats
python scripts/run_rq2.py      # RQ2 skill vs luck
python scripts/run_rq3.py      # RQ3 signal value & calibration
python scripts/run_regime.py   # §9 regime robustness

# 5. Build the full report end-to-end (generates all figures, runs all stats, writes report.docx)
python scripts/build_report.py

# 6. Run the test suite
pytest tests/ -v
```

## Data Note

The raw Twitter scrape is not redistributed due to X's terms of service. The processed file `data/tweets_validated.csv` contains only pseudonymised account identifiers (`account_01` … `account_51`) and derived fields (ticker, sentiment, price outcomes). The original account handles have been removed. To re-run from scratch, you would need to provide the raw scrape in the format described by `data/DATA_DICTIONARY.md`.

## Limitations

See §10 of the report for a full discussion. Key structural limitations: survivorship bias (deleted tweets and deactivated accounts are absent); first-ticker simplification for multi-ticker tweets; LLM classification error not propagated into inference; beats_market assumes frictionless execution and short-selling feasibility; fixed horizon windows; exclusion of neutral tweets; period specificity (2021-2025) and severe data concentration in 2024-2025 (§9.2).
